📞 FREQUENTLY ASKED QUESTIONS (For Presentation)
═══════════════════════════════════════════════════════════════════════════════

Q: Why did you choose Snake over other game ideas?
A: "Snake balances simplicity with complete feature implementation. It showcases
   core game development concepts without unnecessary complexity."

Q: How did you handle memory management?
A: "All memory is allocated with malloc and freed with free. We compiled with
   -fsanitize=address which reported zero errors. The game runs without memory
   leaks at any point."

Q: Why no Raylib graphics version?
A: "We implemented Raylib code to demonstrate graphics capability, but our
   console version is more portable and demonstrates expert C programming. A
   complete console game beats an incomplete graphics version."

Q: How does collision detection work?
A: "We check two conditions: (1) Does the snake head go out of bounds? 
   (2) Does the snake head overlap with any body segment? If either is true,
   game over."

Q: What's the hardest part of the project?
A: "Managing the snake body as a dynamic array and implementing the shift
   operation when moving. We also had to handle the non-blocking keyboard input
   for real-time responsiveness."

Q: Can it run on any system?
A: "Yes! The console version runs on Windows, Linux, and macOS with only a C
   compiler. No external dependencies. Just compile and run."

Q: How would you extend it?
A: "Possible extensions: Raylib graphics, score persistence to file, power-ups,
   obstacles, network multiplayer, but we chose to do one thing perfectly
   rather than many things incompletely."
